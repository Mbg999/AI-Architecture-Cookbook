# Backup & Recovery

Purpose
- Define backup frequency, retention, and recovery RTO/RPO for critical data.
- Scope: databases, object storage, configuration, and stateful services.

Recommendations
- Maintain automated backups with tested restores.
- Keep at least two geographic copies for critical data.
- Define Recovery Time Objective (RTO) and Recovery Point Objective (RPO) per data class.
- Document restore procedures and test them quarterly.

Testing
- Regularly run recovery drills and record timings.
