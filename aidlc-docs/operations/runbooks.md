# Runbooks (Incident Playbooks)

Purpose
- Provide step-by-step remediation actions for common incidents.
- Scope: Pager response steps, mitigation, escalation, and post-incident review.

Template Runbook
- Title / Service
- Symptoms
- Impact
- Immediate mitigation steps
- Diagnostic commands and logs to check
- Escalation contacts
- Recovery steps
- Post-incident actions (postmortem link)

Examples
- High error rate — graceful degradation, rollback, enable feature-flag
- Database connection failures — failover, increase replicas, open support ticket
